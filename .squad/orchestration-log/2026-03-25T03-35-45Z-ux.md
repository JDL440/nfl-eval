## Key Findings

**Three Recurring Shared Seams:**

1. **Shell/nav seam** — `src/dashboard/views/layout.ts` drives every page, but header behavior is styled only through generic `.site-header`, `.header-inner`, `.btn-header`, and `.env-badge`. Mobile hook classes exist in markup/tests but not in stylesheet.

2. **Data-surface seam** — `uns.ts`, `memory.ts`, `config.ts` present operator data differently on small screens (`.runs-table-wrap`, raw `.memory-table`, raw `.artifact-table`) with no shared responsive strategy.

3. **HTMX fragment seam** — Article live partials, runs, memory, publish workflow all swap inner fragments; mobile structure must live in shared fragment classes/CSS, not only page wrappers.

## Recommendations

Treat dashboard mobile as one **shared system change**, not page-by-page cleanup:

1. **UX** defines shared mobile contract (header/navigation, action/filter groups, responsive data-surface, preview chrome)
2. **Code** implements in shared CSS/layout seams (layout.ts, styles.css, fragment renderers)
3. **Code + UX** add tests verifying both markup hooks and stylesheet rules
