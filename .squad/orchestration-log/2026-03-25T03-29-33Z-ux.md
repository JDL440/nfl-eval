## Key Findings

**Root Cause:** Dashboard is a desktop-first system, not a collection of isolated page bugs.

**System-Level Issues:**
1. Shared shell (`layout.ts`) has no responsive nav; fixed 56px header locks across all pages
2. Responsive CSS (`styles.css`) has one small `@media (768px)` block; missing nav/table/action patterns
3. Tables rely on desktop width; no card/stacked fallbacks for `/runs`, `/memory`, `/config`
4. HTMX/SSE fragments inherit desktop markup and bypass mobile context
5. Test coverage protects route behavior but not mobile layout, breakpoints, or touch targets

**Minimum System Change Set:**
- Add shared mobile shell contract (responsive nav pattern)
- Create three shared responsive primitives (action-stack, data-surface, secondary-panel)
- Scope overloaded CSS selectors (e.g., `.agent-grid` collision)
- Ensure HTMX fragments honor mobile classes
- Add structural test assertions for mobile layout and touch targets

**Implementation Split:**
- **UX (days 1–2):** Define mobile shell/nav behavior, shared responsive patterns, selector map
- **Code (days 3–7):** Shell implementation, CSS system, page overrides, regression tests
