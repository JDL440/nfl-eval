## Key Findings

1. Dashboard routes have no auth middleware
2. No session/user persistence in current schema
3. Config subsystem can cleanly absorb auth settings
4. Tests assume unauthenticated direct access

## Recommendation

Implement single-operator local login using:
- Hono middleware for request gating
- SQLite `dashboard_sessions` table for state
- Config-driven auth mode (`off|local`)
- httpOnly session cookies
