# Orchestration Log — UX Agent

**Session:** Stage Badge Mismatch Investigation  
**Date:** 2026-03-23T22:44:04Z  
**Work:** Dashboard Stage Runs badge investigation and decision

## Summary

UX investigated dashboard badge display mismatch: `renderArticleMetaDisplay()` used `article.current_stage` while `renderStageRunsPanel()` rendered `r.stage + 1` as a target stage. This created visual inconsistency where the article header badge and Stage Runs panel showed different stage numbers for the same article.

## Findings

- Article header badge: Displays `article.current_stage` (e.g., "Stage 5 — Article Drafting")
- Stage Runs panel (pre-fix): Displayed `stage + 1` from persisted run record (e.g., "Stage 6" when stored stage was 5)
- Root cause: Mixing of persisted stage value with view-layer transformation

## Decision

Render Stage Runs using the persisted `stage_runs.stage` value directly, without transformation. This aligns with repository contract and eliminates the mismatch.
