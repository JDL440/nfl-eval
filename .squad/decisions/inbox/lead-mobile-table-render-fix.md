# Decision: Mobile Table Render Right-Edge Clipping Fix

**Date:** 2026-07-26
**Author:** Lead
**Status:** Implemented

## Problem

Mobile table PNG images (generated by `.github/extensions/table-image-renderer/renderer-core.mjs`) were clipping text on the right edge. Affected any table with dense text in the rightmost column — visible on the Emmanwori article's "Tier vs What It Requires" and "Position Need vs Why" tables.

## Root Cause

Two interacting issues:

1. **`border-collapse: collapse` + `table-layout: fixed`** — Chrome's collapsed border model can produce subpixel column-width rounding that makes the table 1-4px wider than its container.
2. **Zero-margin `overflow: hidden` on `.table-frame`** — The table filled the frame's content box exactly, so even 1px of overflow from (1) clipped the rightmost column's text.

## Fix (3 parts)

1. **Added `tableFramePaddingX` to layout constants** (4px for mobile, 0 for desktop). This creates a safety buffer inside `.table-frame` between the border and the table. The `overflow: hidden` clips at the padding-box, which now extends beyond the content area.
2. **Changed `border-collapse: collapse` → `border-collapse: separate; border-spacing: 0`** — eliminates Chrome's collapsed border rounding issue while maintaining identical visual appearance.
3. **Increased mobile canvas width by `2 × tableFramePaddingX`** — compensates for the added frame padding so the effective table content area stays the same.
4. **Updated `estimateRowHeight` and `estimateHeaderRowHeight`** — now account for frame border and padding in usable-width calculations.

## Impact

- All future mobile table renders will have the safety buffer.
- Desktop renders are unaffected (`tableFramePaddingX: 0`).
- Existing mobile images need re-rendering to pick up the fix (Emmanwori images re-rendered as part of this change).
