## Key Findings

### Critical Gaps
1. **Model enforcement:** `.squad/config/models.json` documents model assignments but is NOT validated at spawn time. Agents can override.
2. **Token tracking:** No DB schema for LLM token counts. Max tokens are not passed to `task()` calls.
3. **Extension instrumentation:** Substack publisher, Gemini image gen, and table renderer lack usage logging.
4. **Metadata persistence:** Stage transitions log agent name + timestamp but NOT model or cost.

### Ready-to-Implement Surfaces
- `stage_transitions.notes` field can store JSON metadata without schema change (Phase 1)
- `PipelineState.advance_stage()` can accept optional metadata dict
- Gemini extension already captures API response (line 73–85) — just needs logging
- Substack extension tracks image upload count (line 138) — just needs DB write
